Class inheritance

We’re going to be adding bullets into our game, which (in our case) happen to be round, move at an angle, and collide with other sprites. Sound familiar? In fact, the bullets we’ll be creating are going to share a majority of the code from our ball class. In Python we can use class inheritance to “copy” the code from our Ball class into our new Bullet class. We’ve already used inheritance to create our Ball class from the pygame.sprite.Sprite class, but now you’ll see the benefits first hand.

Our Bullet class isn’t going to be exactly the same as our Ball class though. After inheriting from the Ball class we can make our changes using method overriding. So what changes do we need to make? The only real difference is in how we update the class. We don’t want to bounce the bullets around the screen. Instead, when a bullet and ball collide, we want to destroy both. Let’s refactor some of our Ball class to make these changes without duplicating code.

Exercise #1 - http://pastebin.com/Pp50qVCZ

The first thing we refactor is the move functionality in our update method. Our bullets are going to move exactly as our balls do, so we can reuse all of our code by moving it into a move method.

Exercise #2 - http://pastebin.com/9bGRaYHv

Next, we move all of our collision checking code into it’s own method. Remember, we still want to check for collisions in our Bullet class, but we don’t need to bounce every time one is detected. By moving this code, we’ll be able to share all the collision logic between classes, but handle the results differently.

Challenge Problem

Create a rudimentary scoring system in your game.

You can accomplish this by some combination of counting how many bullets have been shot and how many balls have been destroyed. Once all balls have been destroyed you can divide the balls destroyed by bullets fired to get a final score between 0 and 1, where 1 is a perfect score.
