Bouncing revisited

The logic

When figuring out how to bounce a ball, we start with reversing its direction. PyGame coordinates start at zero in the top left and increase as we get further away from the origin.

Because of this, we need to increase the x position when moving to the right, up until the x position is outside the window boundaries. To move to the left, we decrease the x position until we hit zero, which is the beginning of the window.

And remember, we can’t simply check if our box’s position is outside the window and decrease the speed. This is because once the box moves even slightly back into the window it will proceed moving right, and outside the window again.

Instead, we need to “toggle” the direction. Making the box move either right or left and permanently changing the direction until we detect it’s gone outside the window again. To create this toggle with minimal and re-usable code, we use some simple math to reverse the direction of the speed with: x_speed = -x_speed.

PyGame has some helper functions to make coding easier for us. One of them is pygame.Rect. Using this function will create a tuple for us, much like we created a Rect tuple last week, along with some handy move and collide functions. You can see everything pygame.Rect has to offer by checking the documentation at http://www.pygame.org/docs/ref/rect. Let’s write some code to bounce a box left and right.

Exercise #1 - bouncing.py

import pygame, sys

from pygame.locals import *

pygame.init()

DISPLAYSURF = pygame.display.set_mode((640, 480))

pygame.display.set_caption('Bouncing Box')

# Set our color constants

BLACK = (0, 0, 0)

WHITE = (255, 255, 255)

# Set the box properties

box = pygame.Rect(0, 200, 100, 50)

speed = [1, 0]

# Game loop

while True:

    # Handle events

    for event in pygame.event.get():

        if event.type == QUIT:

            pygame.quit()

            sys.exit()

    # Update game state

    box = box.move(speed)

    if box.right >= DISPLAYSURF.get_width() or box.left <= 0:

        speed[0] = -speed[0]

    # Draw screen

    DISPLAYSURF.fill(BLACK)

    pygame.draw.rect(DISPLAYSURF, WHITE, box)

    pygame.display.update()

Adding a dimension

If we want the box to bounce up and down, as well as side to side, we need to make it move in that direction and bounce if it’s gone too far. To make it move in the y dimension, we need to change the second value in our speed list to 1. Your speed variable declaration should now look like this: speed = [1, 1].

We also need to change the y direction when our box has hit the top or bottom of the window. An if statement very similar to our left-right bounce check can accomplish this. Add the highlighted code to your file in the same place you see it appear below.

Bouncing ball

Take some time and try to make your box into a ball. If you completed the challenge problem from last week, you may remember how this is done. The first thing you can do is change the “box” variable to “ball”, along with all its instances in your code. Next, you should change your ball’s Rect so that the width and height match. Afterall, a circle is symmetrical. Then you should draw a circle instead of a rectangle. Consult the documentation at http://www.pygame.org/docs/draw for drawing a circle.

There’s a couple things to note when drawing a circle. You’ll want to use the center of your ball for the position, instead of the top left x and y. PyGame Rects have a handy variable that will keep track of the center for you. Also, since the radius of a circle is half its width, you’ll want to divide the ball’s width in half when passing in the radius.

Once you have a ball, go ahead and customize your game a little. Try changing the colors of the background and your ball. Change the size of your ball and try making it go faster.

Classes

Having a ball bouncing around the screen is great, but what about if we want two balls bouncing around? We’d then have to duplicate all of our ball code. This doesn’t sound that bad for adding just one more ball, but it quickly becomes unwieldy with 5, 10, 20 or more balls. As programmers, our goal is to write less, more maintainable code, as opposed to duplicating the same piece of code. If only there was a way to write all this logic code once and easily create more than one ball with it… Well there is! We do this using a class.

Classes allow us to write modular code once and instantiate it multiple times. Instantiation is a big, complicated sounding word that essentially means “to create an instance of”. So we write code that doesn’t actually do anything until we call it, which creates a new instance of that class. This code can be called many times, which means we can easily create many bouncy balls. Create a new file with the following code. Try typing out the actual Ball class, but feel free to copy and paste the rest, since it’s nearly the same code from before.

Exercise #4 - ball-class.py

import pygame, sys

from pygame.locals import *

pygame.init()

DISPLAYSURF = pygame.display.set_mode((640, 480))

pygame.display.set_caption('Bouncing Ball with classes')

# Set our color constants

BLACK = (0, 0, 0)

WHITE = (255, 255, 255)

# Create the ball class

class Ball():

    # This code gets executed as soon as we create a new instance

    def __init__(self, color, x, y, radius, speed):

        self.color = color

        self.rect = pygame.Rect(x, y, radius * 2, radius * 2)

        self.speed = [speed, speed]

    # Update our game state by moving and bouncing if needed

    def update(self):

        self.rect = self.rect.move(self.speed)

        if self.rect.right >= DISPLAYSURF.get_width() or self.rect.left <= 0:

            self.speed[0] = -self.speed[0]

        if self.rect.top <= 0 or self.rect.bottom >= DISPLAYSURF.get_height():

            self.speed[1] = -self.speed[1]

    # Draw our ball to the screen

    def draw(self):

        pygame.draw.circle(DISPLAYSURF, self.color, self.rect.center, self.rect.width / 2)

# Create a new Ball instance named 'myball'

myball = Ball(WHITE, 100, 100, 25, 1)

# Game loop

while True:

    # Handle events

    for event in pygame.event.get():

        if event.type == QUIT:

            pygame.quit()

            sys.exit()

    # Update game state

    myball.update()

    # Draw screen

    DISPLAYSURF.fill(BLACK)

    myball.draw()

    pygame.display.update()

We’ll go over how this class works in a bit more detail next week. For now, you can see that we have three functions within our class. One occurs as soon as we create a new instance of this class (every time we make ball) and the others are called later in our game loop.

You can see how we instantiate a ball by looking at the “myball” variable declaration. It looks just like calling a function, except we must store it in a variable to use the instance. We pass in a few parameters to this class, just like we would to a function. The parameters in order are: color, x, y, radius, speed. Try using this syntax to create another ball variable. Remember to give it a different x/y position so that it won’t appear directly overtop your first ball. Also, remember to call the update() and draw() functions for that ball instance.
