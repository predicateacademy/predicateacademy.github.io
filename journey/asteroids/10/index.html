Limiting bullets

You can limit the amount of bullets the Ship can fire at any given time to improve gameplay or increase the game’s difficulty.

Exercise #3 - http://pastebin.com/LBsu0HQ2

In this exercise we limit the bullets by creating two variables in our Ship class, one to set to total amount of bullets that can be fired, and another to track how many bullets are currently being fired. We then increase the fired bullets variable each time a bullet is shot and decrease the variable every time one of those bullets has been destroyed.

Resetting lives

You don’t have to have the player’s lives carry over to the next level. Instead, you can reset the player’s lives each level. Resetting the player’s lives could mean setting it to the same number each new level, adding an extra life or two each time the player completes a level, or even defining a different number of lives for each level in your levels.py and reading that value. Here are some tips to accomplish this change:

Ensure you create the player variable before adding it to the Sprite Group.

We did this last week when adding text into the game. If you didn’t complete this exercise, you’ll at least want to make the first change from week 9, exercise #5.

Find the code where you check if the level is complete, and set the player’s lives variable there.

There are two pieces of code which use the level_complete variable. The first is where we set the variable to either True or False, and the second is where we act on that deduction. The second is where you’ll want to set or reset the player’s lives.

Rotate bullets

If your bullet sprite isn’t round, then you may have noticed that when firing in any direction besides up will cause your bullets to be tilted. There is a simple fix for this behaviour by rotating the bullet sprite according to the angle it travels at. Here’s some tips for fixing this bug:

Look inside of your Ship’s “turn” method for the line where the Ship is rotated.

You’ll want to use this same function to rotate your bullet sprite.

Rotate the bullet sprite as soon as it’s created, inside the init method.

Since all bullets travel in just one direction, you can rotate the object once and it will look correct throughout it’s time on screen.

Subtract the bullet’s angle by starting angle of your Ship to point in the correct direction.

Remember, the ship starts by facing upwards, and therefore at an angle of 90.
