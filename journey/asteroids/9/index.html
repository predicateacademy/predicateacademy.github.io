Bug fixes

Before we get into making some improvements to our game, let’s fix a couple bugs in our code.

Exercise #1 - http://pastebin.com/LhTxTK5E

In this round of bug fixing, we first ensure that the balls will only bounce off of other balls and not bullets. Then we set the Ship class to use the x/y parameters as the center of the ship. Enforcing this consistent use of the x/y position for the center of our objects will prevent some future bugs. We next make sure that our ship will only collide with other ball objects and not the text objects we’ll be creating shortly. Finally, we take advantage of the angle parameter in our Ball class and randomize all of the balls directions.

Improvements

Sounds

The first improvement we’ll make to our game is adding sound to some game actions.

Exercise #2 - http://pastebin.com/G7etCRRF

Adding sounds to our game doesn’t require much code as seen in this exercise. We just need to set our class sound files and play them at the appropriate time. You can take this further and add sounds to events like ball boundary collisions and collisions between balls.

Text

We can add text to our game to introduce new levels, display the players lives/score, or end the game with a “game over”. Adding text only requires a few lines, but in order to robustly create and remove text to the screen we’ll create a Text class.

Exercise #3 - http://pastebin.com/G05EadRP

You can see our Text class isn’t too complicated. We simply take in all the required parameters like the text, size, color, etc., and render the font. The interesting part is the update method. To make displaying and deleting text much easier, we provide a duration parameter in which we declare how many frames the text will be displayed for.

Exercise #4 - http://pastebin.com/q1eWaRvK

In this exercise, we begin taking advantage of our newly created Text class. We do this by displaying the level number for 120 frames (roughly 2 seconds), as well as clearing the screen and displaying “GAME OVER” when a loss occurs.

Exercise #5 - http://pastebin.com/pCKEGXis

Our final use of the Text class is displaying the amount of lives our player has remaining.

More bug fixes

Having sounds and text in our game is great, but there are still some bugs in need of fixing.

Exercise #6 - http://pastebin.com/uBzDyYeK

The first bug we fix here is the ability for our ship to move outside of the game window. We do this by saving the ship’s position and resetting the x or y position if the ship has gone outside the boundaries. Then we prevent the game from crashing every time you win and instead display the text “YOU WIN”.

Exercise #7 - http://pastebin.com/j22YT9tX

In this exercise, we prevent balls from spawning on top of each other (or the ship). We do this by generating a random position for each ball, then checking if it collides with any other sprite. If it does collide we repeat the process and generate a new position, while if it does not collide we use those coordinates for the ball.

Boss levels

We can have more control over the difficulty of our levels by adding multi-hit balls. These balls will be able to take more damage than just a single hit and can act as a “boss level”.

Exercise #8 - http://pastebin.com/ARb4Km6M

Here we give each ball an optional damage parameter. Then we update our Ship and Bullet classes to use the damage variable instead of an arbitrary value of 1.

Exercise #9 - http://pastebin.com/UL9iJ97Z

Finally, we add a damage value to our levels.py file for the game to use.
