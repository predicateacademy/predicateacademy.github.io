Web - Week 1

Intro

Language(s) of the web

In all previous Predicate classes, we’ve used Python as our programming language of choice. We’ve created anything from LED controllers and Minecraft Simon, to robot AI and video games. But one place we can’t use Python, is inside our internet browser. And instead of using just one language in the browser, most of the web is powered by three: HTML, CSS, and JS.

The first language we’ll be exploring is HTML, or Hypertext Markup Language. When looking at a web page, HTML defines what will be displayed, or the content of the web page. The next language, CSS, stands for Cascading Style Sheets. CSS defines how and where the content will appear. Finally, we have JavaScript (JS for short), which defines all the logic on our webpage and is the most akin to Python of the three.

Getting started

HTML

Let’s get started with HTML by creating a very basic “Hello, world!” webpage. First, create a “predicate-web” directory in your home directory. This is where we’ll store all of the files for this class.

Exercise #1 - http://pastebin.com/P4ZPzy1G

Save this exercise as “index.html” and place it in your “predicate-web” directory. Opening this file in your web browser (Ctrl + O), you can see that we get a very basic page with the text “Hello, world!” and a title of “My Website”.

Now let’s discuss the code in this exercise. The HTML file starts with <!doctype html>, which declares what version of HTML we’re going to be using. Luckily, from HTML5 onwards, we only need to use “html”. Previous versions of HTML required some long version numbers and URLs. Next in our file is <html>. Because HTML is a markup language, we use tags to create the structure of our web page. These tags consist of our desired elements name surrounded by “arrows” (< and >). This first <html> is an “opening” HTML tag. If we look at the very end of our document, we can see the closing tag </html>. Everything between the opening and closing element is considered a “child” of the HTML element, meaning it is contained within it.

Continuing down our file, we come to the head element. This element contains all of the information, or metadata, about our page. None of this information is directly seen on the page. The only element contained within our head is the title. As you may guess, this element sets the title you see in tab bar or the window title.

Finally, we get to the body element. This is where we declare all of the elements we want to appear on our page. Currently we just have some plain text, but you almost always want to put your text inside of another element. Let’s explore some of those elements now.

Exercise #2 - http://pastebin.com/WGbYTWcN

Here we introduce two new elements. The first, h1, stands for “header 1”. There are six header elements in HTML numbered 1 through 6. You can change your header to another, just remember to change both the opening and closing tags. Then we have the p or paragraph element. Using these elements will provide some spacing styles for us. However, we don’t have to depend upon the built-in styles alone, we can define our own.

CSS

Now that we’ve covered the basics of HTML, let’s dive into CSS. To make styling our page a little easier, we’re going to keep our CSS in a separate file. Go ahead and create a file named “style.css” in your predicate-web directory, then complete this exercise.

Exercise #3 - http://pastebin.com/a1tDYUnt

We use a link tag to point the web page to our CSS file. The rel and href parts are attributes of the link element, and provide valuable information. Rel defines what type of link it is and href stands for hypertext reference, which means the location of our file. This alone won’t change anything though, so let’s start writing some CSS.

Exercise #4 - http://pastebin.com/5M5ggu14

In this exercise we create our first bit of CSS. As you can see, CSS does not use the same markup/tag syntax that HTML employs. Instead, CSS uses a property: value syntax.

Starting from the beginning of the file, we use the “body” keyword to declare what element type we’re going to style. Body could be replaced with any element inside of your document (excluding head) to target that element. After the element name, we use curly brackets {} to contain all the styles we want to apply. Finally, inside of the curly brackets we use a property name, followed by our desired value to change that style.

Try altering the values of the styles in your CSS file. You can find more CSS color names by Googling “CSS Color Data Types”. Also try moving the styles around, maybe setting the color of your p elements, or the background of your header.

Servers

Right now we’re viewing our web page as a file on our system, but that’s not usually how you view a website. Instead, you type in the website address and a server returns the HTML to your browser. So let’s setup a server to view our web page from.

To start a server, first open up a terminal and cd into the “predicate-web” directory. Then all we have to do is type “python3 -m http.server 8080”. You should get a printout back saying “Serving HTTP on 0.0.0.0 port 8080 …”. Then, head to your web browser and go to the address “http://0.0.0.0:8080/”. You’re now viewing your website from your own server!

Links

Earlier in this lesson you learned that HTML stands for Hypertext Markup Language. Well what is hypertext? Hypertext is any text that contains hyperlinks. Hyperlinks, in effect, are pieces of text that point to another location. Whether it be another place on the page or another website, they point to a different location. Since links are so critical to HTML, let’s learn how to create our own.

Exercise #5 - http://pastebin.com/3keW4TEJ

Here we created two links, one to Google and the other to Predicate’s own website. We did this using anchor elements. Just like before, the href attribute equals the location of our anchor. In-between the opening and closing tags of the anchor element is where we put the text we want to be visible.

You’ll also notice a new br element between the two links. Br means line break, and separates elements onto their own line. Try removing the br and see what happens.

Exercise #6 - http://pastebin.com/emwQnEGp

Now that we have links on our page, we can begin to style them. In this exercise, introduce the “text-decoration” CSS property in order to remove the underline from our anchor elements. Text decoration is also useful for adding an underline, overline, or line-through.

In the second part of this exercise, we target all a elements again, but add on “:hover”. CSS supports styling of certain element states, which hover is one of. This means that whenever we hover over the targeted element, these styles contained within will apply. In our file, we add the underline back in when the links are hovered atop.
