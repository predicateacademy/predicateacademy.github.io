Obstacles

The first type of obstacle we’ll introduce are holes. As you’d expect, these holes will be gaps in the ground object that you’ll need to jump over. You’ll need a separate image for these holes, which you can create yourself to fit your game.

Exercise #4 - http://pastebin.com/FKyyQiuV

Next, we’ll add sprite based obstacles to our game. These obstacles can be anything you want, from a static enemy player, to spikes on the ground. For each new obstacle you add, you’ll want to load in the image and pass it into your ground object.

Exercise #5 - http://pastebin.com/wB0nF8GM

Gameplay tuning

At this point your game is near feature complete, so you’ll want to start tuning the gameplay. You may first want to scale all your images to the sizes you’re using in-game to achieve maximum performance. Then start customizing this short list of gameplay features:

  • Game speed
  • Tile speeds
  • Obstacle sizes
  • Hole length
